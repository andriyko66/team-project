# ADR 0003: API Style and Error Model

## Статус
Прийнято

## Контекст
Проєкт надає HTTP API для роботи з каталогом товарів.
API використовується клієнтською частиною (браузер) та описується
через OpenAPI (Swagger).

Важливо мати єдиний стиль API та однаковий формат помилок,
щоб клієнту було легко обробляти відповіді.

## Рішення

### Стиль API
API реалізується у стилі REST з такими правилами:
- використовуються HTTP-методи: GET, POST, PUT, DELETE
- ресурси описуються іменниками у множині (наприклад, /items)
- HTTP-статуси відповідають результату операції

### Статуси відповідей
- 200 — успішне отримання або оновлення
- 201 — успішне створення
- 204 — успішне видалення без тіла відповіді
- 400 — помилка валідації даних
- 404 — ресурс не знайдено
- 500 — внутрішня помилка сервера

### Формат помилок
Усі помилки API повертаються в єдиному форматі:

```json
{
  "error": "ErrorType",
  "message": "Опис помилки"
}
