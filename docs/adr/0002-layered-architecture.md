# ADR 0002: Layered Architecture

## Статус
Прийнято

## Контекст
Проєкт є навчальним і розробляється однією людиною.
На поточному етапі важливо мати просту та зрозумілу структуру коду,
яка дозволяє легко розширювати функціональність.

## Рішення
Обрано шарову архітектуру (layered architecture) у вигляді модульного моноліту.

Використовуються такі шари:

- api/ — обробка HTTP-запитів і відповідей (контролери, роутери)
- service/ — бізнес-логіка та use-cases
- domain/ — доменні сутності та їх правила

## Правила взаємодії шарів
- api звертається тільки до service
- service звертається до domain
- api не звертається напряму до domain або бази даних
- domain не залежить від api та service

## Наслідки
- Код стає зрозумілим і структурованим
- Легше підтримувати та розширювати проєкт
- Архітектура може бути розширена у майбутньому
